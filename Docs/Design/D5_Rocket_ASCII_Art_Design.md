## 1. 디자인 원칙 및 제약사항

### 1.1 총괄PD 디자인 원칙 3대 반영

| 원칙 | ASCII 아트 반영 방법 |
|------|---------------------|
| **실제 로켓 발사 연상** | 클래스별 외형이 실제 로켓 라인업(소형 관측~Saturn V급)을 연상시키도록 설계. 단 수, 부스터 유무, 페어링 형태 등 실제 구조 반영 |
| **극명한 희비교차** | 성공 시 불꽃/글로우 프레임 → 깨끗한 상승, 실패 시 점진적 경고 → 극적 폭발/분해/추락. 컬러(`--green` vs `--red`)로 감정 대비 극대화 |
| **목표 가시성** | 6개 클래스가 나란히 비교 가능하도록 동일 중심선 기준 정렬. 크기 차이가 한눈에 보임. 단 수/부스터 수로 성능 직관적 표현 |

### 1.2 기술 제약

| 항목 | 제약 |
|------|------|
| **폰트** | `'Share Tech Mono', monospace` — 모노스페이스 전제 (6.6px/char) |
| **CSS 변수** | `--green: #00e676`, `--red: #ff1744`, `--amber: #ffab00`, `--cyan: #00e5ff` |
| **발사 오버레이** | `launch-rocket-pre` 요소 내부, `text-align: center` |
| **기존 건물 너비** | 14~18자 (참고용, 로켓은 발사 오버레이 전용이므로 별도) |
| **한글/CJK** | 이름 라벨에 CJK 포함 시 `_padEndVisual()` 사용 필요 |
| **기존 코드 호환** | 현재 `getRocketArtHtml()` (game-state.js L1260~1323)의 구조를 기반으로 확장 |

### 1.3 크기 설계 체계

각 클래스의 ASCII 아트 크기는 실제 로켓 질량 비례를 압축한 로그 스케일로 설계한다.

| 클래스 | 실제 질량 (kg) | 가로 (chars) | 세로 (lines) | 비고 |
|--------|---------------|-------------|-------------|------|
| 베가 (VEGA) | 850 | 15 | 12 | 최소형, 단순 구조 |
| 아르고 (ARGO) | 3,200 | 19 | 16 | 소형 + 스러스터 |
| 헤르메스 (HERMES) | 12,000 | 21 | 22 | 2단 + 페어링 |
| 아틀라스 (ATLAS) | 45,000 | 23 | 28 | 재사용 1단 + 착륙 다리 |
| 셀레네 (SELENE) | 120,000 | 27 | 34 | 3단 + 측면 부스터 |
| 아르테미스 (ARTEMIS) | 350,000 | 31 | 40 | 초대형 완전재사용 |

## 2. 로켓 모델별 ASCII 아트 (정지 상태)

### 2.1 베가 (VEGA) — 프로토타입 단발 로켓

고도 10km 도달 목표. 동체+탱크+엔진 3부품 최소 구성. Falcon 1 / 관측용 소형 로켓 연상.

```
       *
      /|\
     / | \
    / VEGA \
   /________\
   |  [NAV] |
   | .----. |
   | | SYS| |
   | '----' |
   |[ENGINE]|
   |________|
   /________\
     | || |
```

- **가로**: 15자
- **세로**: 13줄
- **특징**: 단일 동체, 노즈콘 최소화, 1기 엔진 노즐
- **시각적 키워드**: 가늘고 짧은 연필형 로켓

### 2.2 아르고 (ARGO) — 스러스터 강화형

카르만 선(100km) 도달 목표. 측면 스러스터 추가. 소유즈 초기형/블루오리진 연상.

```
         *
        /|\
       / | \
      / ARGO \
     /________\
     | [PAY]  |
     | .----. |
     | | NAV| |
     | | SYS| |
     | '----' |
     |[AVION] |
     | O  O   |
     |--------|
     |[PROP]  |
    /| 78 kN  |\
   / |________| \
  /  /________\  \
  |  |  || |  |  |
  T  | || || |  T
```

- **가로**: 19자
- **세로**: 18줄
- **특징**: 좌우 `T` 스러스터 마커, 이중 벽 구조, 확장된 추진부
- **시각적 키워드**: 측면 보조 추진기가 달린 세장형 로켓

### 2.3 헤르메스 (HERMES) — 2단 궤도 진입 로켓

저궤도(LEO) 진입. 2단 + 페이로드. Falcon 9 2단부 연상.

```
           *
          /|\
         / | \
        /  |  \
       / HERMES\
      /_________\
      |[PAYLOAD]|
      | .-----. |
      | | NAV  | |
      | | SYS  | |
      | '-----' |
      |         |
      |[AVION]  |
      | O  O  O |
      |  GYRO   |
      |_________|
     =====2=====    <-- 단분리 라인
      |[PROP-2]|
      | .-----. |
      | |310kN| |
      | '-----' |
      |_________|
     /___________\
       |  ||  |
      /|  ||  |\
     /_|__||__|_\
```

- **가로**: 21자
- **세로**: 25줄
- **특징**: 2단 구조, 명확한 단분리 라인(`=====2=====`), 3기 자이로, 확장 페이로드 베이
- **시각적 키워드**: 키가 큰 2단 로켓, 중간에 분리 볼트 표시

### 2.4 아틀라스 (ATLAS) — 재사용 중형 로켓

중궤도(MEO) 진입. 1단 재사용 구조 + 착륙 다리. Falcon 9 Block 5 연상.

        |  .-------.  |
        |  | O O O |  |
        |  | GYRO  |  |
        |  '-------'  |
        |_____________|
       ======S2======    <-- 2단 분리
        | [PROP-2]   |
        |  .-------.  |
        |  |1200kN |  |

- **시각적 키워드**: SpaceX Falcon 9의 착륙 다리가 달린 슬림한 재사용 로켓

- **시각적 키워드**: Saturn V급 초대형 로켓, 좌우 대칭 부스터

## 5. 로켓 클래스별 실패 차이

### 5.1 클래스별 실패 스케일 매핑

각 로켓 클래스의 폭발/분해 규모는 크기와 연료량에 비례한다.

| 클래스 | 폭발 규모 | 잔해 범위 | 프레임 속도 | 화면 효과 |
|--------|----------|----------|------------|----------|
| **VEGA** | 작음 (5줄) | 좁음 (12자) | 빠름 (500ms/프레임) | 작은 불꽃 → 빠른 소멸 |
| **ARGO** | 소형 (7줄) | 보통 (16자) | 보통 (600ms/프레임) | 스러스터 파편 추가 |
| **HERMES** | 중형 (9줄) | 넓음 (20자) | 보통 (650ms/프레임) | 2단 분리 잔해 산란 |
| **ATLAS** | 대형 (11줄) | 넓음 (24자) | 느림 (700ms/프레임) | 착륙 다리 파편, 대형 화염구 |
| **SELENE** | 초대형 (14줄) | 매우 넓음 (30자) | 느림 (750ms/프레임) | 부스터 분리 + 다중 폭발파 |
| **ARTEMIS** | 극대형 (16줄) | 최대 (35자) | 매우 느림 (800ms/프레임) | 33기 엔진 연쇄 폭발, 화면 전체 진동 |

- ASCII 아트 문자열은 사전 생성(빌드타임)하여 런타임 문자열 연결 최소화
- 프레임 전환은 `textContent` 교체 (innerHTML 아닌) — XSS 방지 + 성능
- CSS 애니메이션은 `transform` + `opacity` 위주 (GPU 가속, reflow 방지)
- 총 프레임 수: 성공 시 최대 22프레임(8단계 x 평균 2.75), 실패 시 최대 3~4프레임
- `requestAnimationFrame` 사용 대신 `setTimeout` 유지 (기존 코드 호환, 정밀 타이밍 불필요)
