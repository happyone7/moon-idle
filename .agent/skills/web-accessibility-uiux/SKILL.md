# Web Accessibility UIUX

Use this skill for accessibility and interaction quality reviews.

## Checklist

1. Keyboard and focus
- Every critical action reachable without mouse.
- Visible focus ring and logical tab order.
2. Semantics and labels
- Buttons/inputs have accessible names.
- Dynamic status changes announced when needed.
3. Visual accessibility
- Color contrast for text and controls.
- No color-only state communication.
4. Motion and comfort
- Respect reduced motion preference.
5. Mobile ergonomics
- Adequate touch target size and spacing.
